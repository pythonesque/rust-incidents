Represents an error.

At the very least the name part of an error needs to be implemented. Everything else is pretty much optional but it's recommended to implement as much as possible to aid the user for debugging. Note that a lot of information can be computed lazily.

Required Methods

fn name(&self) -> &str

Provided Methods

fn description(&self) -> Option<&str>

An optional description of the error.

fn detail(&self) -> Option<String>

Optional detail information generated from the error's data.

Optionally the cause of an error.
